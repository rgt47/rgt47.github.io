Introduction

This is the first in a series of posts with suggested strategies for leveraging a set of open source technologies to provide solutions to one of the central challenges in the practice of data science i.e. how to effectively communicate analysis results to clients and collaborators. The group of technologies (or stack) we’ll employ is: linux, R, Shiny, Caddy, and Docker.

This initial post provides a minimal example of applying these technologies for the development and hosting an interactive tool for conducting a statistical power analysis.

We start with a simple, but still useful, shiny app and end with a secure (encrypted and authenticated) web site with a custom domain name.

Lets assume we’re just finished developing a ‘shiny’ new shiny app, named power0 .

See the details of the power0 shiny app here.

A screenshot of the finished product shows a shiny app with a widget to select the sample size and a visualization (2D plot) of the power as a function of the standardized effect size:

Photo by Ian Taylor on Unsplash

Here is our dockerfile:

FROM rocker/shiny:4.2.0
COPY power0.R /srv/shiny-server/
USER shiny
CMD ["/usr/bin/shiny-server"]

Thats it. With the dockerfile and the app.R fine in the same directory we can first build the docker image, and then push it to Docker hub as a convenient repository. (Alternatively we could copy it directly to the server or to another site, e.g github.com. )

Hosting

Once we have a production ready dockerized shiny app available, we’ll want to host it on the internet. To do that we’ll need the following :

  1. a virtual server
  2. a static IP address
  3. a domain name
  4. a web server
  5. an SSL certificate (encrypted communication)
  6. an authentication method (password protection)
  7. a reverse proxy method

This can all be done at no cost if you have your own (self-hosted) server and domain name, or at minimal cost using a cloud-hosting service (e.g. Amazon’s EC2 or Digital Ocean) and a “leased” domain name from, e.g. via GoDaddy, or Amazon’s Route 53.

There are a number of cloud based server options: Microsoft Azure, Google Cloud, Amazon AWS, Digital Ocean to name a few.

To construct the web site we need four files in the home directory for default user ubuntu. The first file is power0.R our shiny R program file. We can upload it from our local machine with scp or copy it in from github.

git archive –remote=ssh://host/pathto/repo.git HEAD README.md | tar xO

We’ll use Caddy as our web server To allow the automation of one of the most complicated parts of this exercise, the i.e. aquiring and install an SSL certificate.

The second file is the Caddyfile configuration file. The caddyfile provides three things. 1. the site domain name. 2. the authentication pair login/hash-password, and 3. the ‘reverse proxy’ map that redirects requests to port 443 (ssl port) to port 3838 (shiny port).

The Caddyfile and docker-compose.yml file are

Caddyfile

auth info bob/utter


rgtlab.org {
    basicauth * {
        bob JDJhJDE0JEw4Zk5MQi5nYndEaHRjVTFVeGl2a2VBZTVPcFI0THhEZUd0OEJkOTR2d0VjRUxsMDBsQUg2
    }
    root * /srv
    handle_path /power0/* {
        reverse_proxy power0:3838
    }
    file_server
}

And the third file is the docker compose file that pulls our shiny app, and the caddy server and create a local network for them to communicate in.

docker-compose.yml


version: "3.7"

services:
  power0:
    image: rgt47/power0:latest
  caddy:
    image: caddy:2.3.0-alpine
    ports:
      - "443:443"
    volumes:
      - $PWD/Caddyfile:/etc/caddy/Caddyfile
      - $PWD/site:/srv
      - caddy_data:/data
      - caddy_config:/config
volumes:
  caddy_data:
  caddy_config:
